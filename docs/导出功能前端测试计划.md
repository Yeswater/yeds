# 导出功能前端测试计划

依据 [doc/导出功能需求设计.md](../doc/导出功能需求设计.md) §11、§8 与 `RunQuery.vue` / `api.js` 实现编制。

## 1. 测试范围

| 范围 | 说明 |
|------|------|
| 页面 | 服务运行 → 执行查询后出现结果区 → **服务导出** Tab |
| 接口 | 仅经 `bids-exec`：`/api/runtime/models/{code}/export/*`、`/api/runtime/export/tasks/*` |
| 不在范围 | 直连 `bids-export:8083`、配置态 SQL 编辑 |

## 2. 前置条件

- Docker 栈：`bids-web`(8080)、`bids-exec`(8082)、`bids-export`(8083)、MySQL、RustFS、业务库（如 PostgreSQL `fin_holdings_pg`）
- 账号：`admin` / `admin`（或 `.env` 配置）
- 至少一个 **已发布** SQL 服务，且已能 **执行查询** 成功（结果区可见）

## 3. 用例清单

### 3.1 Tab 与禁用态

| ID | 步骤 | 预期 |
|----|------|------|
| FE-01 | 未执行查询时打开结果区（若存在） | 无结果卡片则无导出 Tab |
| FE-02 | 执行查询成功后切换 Tab | 存在「查询结果」「原始响应」「运行脚本」「**服务导出**」，且顺序在运行脚本之后 |
| FE-03 | 有查询结果但未切到导出 Tab | 「导出」「预估行数」在导出 Tab 内；`canExport` 为真时按钮可点 |
| FE-04 | 刷新页面后仅有表单、无 `result` | 导出按钮禁用（`!result`） |

### 3.2 说明与预估

| ID | 步骤 | 预期 |
|----|------|------|
| FE-05 | 进入「服务导出」Tab | 展示文案：「导出当前表单条件下的**全量**数据，与上方表格分页无关」 |
| FE-06 | 进入 Tab（已实现 watch） | 自动调用 `estimate` + `listTasks`；展示约 N 行及模式标签 |
| FE-07 | 点击「预估行数」 | 再次请求 estimate；`SYNC` 显示「同步 xlsx」；`ASYNC` 显示「异步 zip」；`ASYNC_SUGGESTED` 显示「建议异步」 |
| FE-08 | COUNT 超时场景 | `estimatedRows` 为空时文案：「行数预估超时，建议异步导出」 |

### 3.3 同步导出（≤10000 行）

| ID | 步骤 | 预期 |
|----|------|------|
| FE-09 | estimate 返回 `mode=SYNC`，点击「导出」 | POST `/export`，浏览器下载 `.xlsx`，文件名来自 `Content-Disposition` |
| FE-10 | 修改分页为第 2 页后导出 | 导出数据与表单条件一致，**不受** `currentPage`/`pageSize` 影响 |
| FE-11 | 多选品种（`product_codes_csv` 数组） | 参数以逗号拼接提交，与执行查询一致 |

### 3.4 异步导出（>10000 行或 ASYNC*）

| ID | 步骤 | 预期 |
|----|------|------|
| FE-12 | estimate 为 `ASYNC` 或 `ASYNC_SUGGESTED`，点击「导出」 | POST `/export/tasks`，提示已创建任务 |
| FE-13 | 任务 RUNNING | 每约 3s 轮询，进度条 `progressPct` 更新 |
| FE-14 | 任务 SUCCESS | 出现「下载 zip」；`downloadReady=true` |
| FE-15 | 点击下载 | GET `/export/tasks/{id}/download` 带 Basic 认证；302 后新窗口打开预签名 URL（或直传 zip） |
| FE-16 | 任务 FAILED | 轮询停止；列表/当前任务展示失败状态 |

### 3.5 任务列表

| ID | 步骤 | 预期 |
|----|------|------|
| FE-17 | 进入导出 Tab 或导出完成后 | 下方表格展示当前用户最近 20 条（`limit=20`） |
| FE-18 | 列表行 `downloadReady` | 「下载」链接触发与 FE-15 相同逻辑 |

### 3.6 异常与安全

| ID | 步骤 | 预期 |
|----|------|------|
| FE-19 | 未登录访问 API | 401，前端提示错误 |
| FE-20 | 同步路径预估 >10000 仍点导出（若后端返回 400） | `ElMessage.error` 展示 message |
| FE-21 | `bids-export` 不可用 | estimate/export 失败，前端错误提示明确 |

## 4. 全局注意事项（已写入 `.cursor/rules/bids-export-and-integration.mdc`）

联调/改导出相关代码前请对照项目规则，摘要：

- bids-export：MySQL DDL 语法、sql.init.mode、JDBC 多方言、容器非 Restarting
- 构建：`-Dmaven.test.skip=true`、测试方法 throws
- 前端：Tab 自动预估、参数与 execute 一致、302 下载不用 fetch follow

## 5. 与设计差异（已修复项）

| 项 | 设计 | 原实现 | 处理 |
|----|------|--------|------|
| 进入 Tab 自动预估 | §11 | 仅手动/点导出前 | 增加 `watch(resultTab)` |
| 多选参数 | 与 execute 一致 | 数组未 join | `buildExportParameters` 对齐 |
| 异步下载 302 | exec 302 预签名 | `redirect:follow` 易 CORS 失败 | `redirect:manual` + `window.open` |
| 后端 export 启动 | — | MySQL `CREATE INDEX IF NOT EXISTS` 失败 | `SPRING_SQL_INIT_MODE=never` + init SQL |
| PG 数据源导出 | 方言支持 | 缺 `postgresql` 驱动 | `bids-export/pom.xml` 增加依赖 |

## 6. 建议测试数据

- **同步**：`fin_holdings_pg` + 窄条件（少量行），`mode=SYNC`
- **异步**：放宽条件或调低阈值环境，使 `estimatedRows > 10000`（或依赖 `ASYNC_SUGGESTED`）
