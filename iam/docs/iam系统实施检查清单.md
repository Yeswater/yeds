# YEDS IAM 系统实施检查清单

## 1. 使用说明

- 本清单用于 IAM 项目实施过程的跟踪与验收。
- 建议按阶段执行，完成后在对应项打勾。
- 若涉及跨系统联调，请同步记录问题单与回归结果。

---

## 2. 阶段一检查清单（MVP：先可用）

### 2.1 基础建设

- [ ] 已完成 IAM 项目目录与模块初始化。
- [ ] 已建立基础配置（环境变量、配置中心、密钥配置）。
- [ ] 已建立数据库连接与迁移机制（DDL 可重复执行策略明确）。

### 2.2 数据模型

- [ ] 已创建 `iam_user` 表及索引。
- [ ] 已创建 `iam_role` 表及索引。
- [ ] 已创建 `iam_permission` 表及索引。
- [ ] 已创建 `iam_user_role` 关系表及联合索引。
- [ ] 已创建 `iam_role_permission` 关系表及联合索引。
- [ ] 已创建 `iam_token_session` 表。
- [ ] 已创建 `iam_audit_log` 表（含时间检索索引）。

### 2.3 认证能力

- [ ] `POST /api/iam/auth/login` 已可用。
- [ ] `POST /api/iam/auth/refresh` 已可用。
- [ ] `POST /api/iam/auth/logout` 已可用。
- [ ] `GET /api/iam/auth/jwks` 已可用。
- [ ] JWT 签发参数（issuer、audience、ttl）已配置。
- [ ] Token 吊销机制已生效（黑名单或会话失效策略）。

### 2.4 授权能力

- [ ] `POST /api/iam/authorize/check` 已可用。
- [ ] `POST /api/iam/authorize/batch-check` 已可用。
- [ ] `GET /api/iam/authorize/user-permissions` 已可用。
- [ ] RBAC 基础判定已通过联调验证。

### 2.5 与 APIG 集成

- [ ] APIG 已接入 IAM 的 JWK 拉取。
- [ ] APIG 本地缓存策略已配置。
- [ ] 鉴权失败统一错误码已对齐。
- [ ] Token 过期/非法/吊销场景已验证。

### 2.6 与 BIDS 集成

- [ ] 身份上下文（用户 ID、角色、租户）已透传。
- [ ] BIDS 已调用 IAM 进行权限判定。
- [ ] BIDS 关键操作审计已关联 IAM 身份。

### 2.7 审计与可观测

- [ ] 登录成功/失败事件已落审计。
- [ ] 鉴权允许/拒绝事件已落审计。
- [ ] 审计日志支持按用户和时间查询。
- [ ] 已接入基础监控指标（登录成功率、鉴权拒绝率、接口延迟）。

### 2.8 阶段验收

- [ ] 完成阶段一联调用例回归。
- [ ] 完成阶段一风险评审。
- [ ] 阶段一上线评审通过。

---

## 3. 阶段二检查清单（增强：可治理）

### 3.1 认证增强

- [ ] MFA（短信/OTP）能力可配置启停。
- [ ] 高风险账号已强制启用 MFA。
- [ ] 基础 SSO（OIDC）接入已完成。

### 3.2 授权增强

- [ ] 支持 IP 条件策略。
- [ ] 支持时间窗口条件策略。
- [ ] 支持环境标签条件策略。
- [ ] 条件策略与 RBAC 组合判定已通过验证。

### 3.3 应用身份治理

- [ ] `iam_client` 管理接口已上线。
- [ ] 密钥轮换流程可执行且有回滚方案。
- [ ] 密钥加密存储与脱敏展示已落实。

### 3.4 权限变更流程

- [ ] 权限变更审批流程已上线。
- [ ] 权限变更审计日志可追溯审批单。
- [ ] 紧急授权机制与回收机制已明确。

### 3.5 阶段验收

- [ ] 阶段二能力灰度验证通过。
- [ ] 关键风险项复盘完成。
- [ ] 阶段二上线评审通过。

## 5. 上线门禁清单（每次发布必查）

### 5.1 功能门禁

- [ ] 登录、刷新、登出、吊销用例通过。
- [ ] 鉴权允许/拒绝用例通过。
- [ ] 密钥轮换前后用例通过。

### 5.2 安全门禁

- [ ] 未发现默认弱口令账号。
- [ ] 未发现超权角色配置。
- [ ] 未发现未鉴权可访问接口。

### 5.3 性能门禁

- [ ] 鉴权接口 P95 延迟达标。
- [ ] 峰值流量下错误率达标。
- [ ] 缓存命中率达标。

### 5.4 可观测门禁

- [ ] 关键指标可视化看板正常。
- [ ] 告警策略生效且可通知到人。
- [ ] 审计日志采集完整无明显丢失。

### 5.5 回滚门禁

- [ ] 回滚脚本已验证可执行。
- [ ] 旧版本兼容窗口已确认。
- [ ] 值班与应急联系人已就位。

---

## 6. 发布后巡检清单（T+1）

- [ ] 登录成功率无异常波动。
- [ ] 鉴权拒绝率无异常上升。
- [ ] 审计日志写入延迟在阈值内。
- [ ] 无高优先级生产故障单。
- [ ] 发布复盘纪要已输出并归档。

---

## 7. 责任分工建议

- IAM 研发：认证授权核心能力、数据模型、接口稳定性。
- APIG 研发：网关鉴权集成、缓存与错误码统一。
- BIDS 研发：业务权限点接入、身份上下文消费。
- 安全/运维：密钥治理、告警策略、上线与回滚保障。
- QA：联调用例、回归验证、上线门禁核查。
